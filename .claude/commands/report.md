# report.md

生成项目进度报告或查看特定任务的状态。

## 使用方法
```
/report [任务ID]
```

## 参数说明

- `任务ID` (可选): 指定任务的ID，如 TASK-001
  - 如果提供，显示该任务的详细信息
  - 如果不提供，显示所有任务的汇总报告

## 示例

```
/report              # 显示所有任务的汇总报告
/report TASK-001     # 显示TASK-001的详细状态
```

## 报告内容

### 汇总报告包含
- 任务总数统计
- 各状态任务数量
- 各团队工作负载
- 最近完成的任务
- 待处理的高优先级任务

### 详细报告包含
- 任务基本信息
- 当前状态和负责人
- 各阶段完成情况
- 相关设计文档
- 测试结果
- 审核意见
- 历史记录

## 状态说明

- `pending`: 待开始
- `design`: 架构设计中
- `development`: 开发实现中
- `testing`: 测试验证中
- `review`: 代码审核中
- `revision`: 修改完善中
- `completed`: 已完成
- `cancelled`: 已取消

## 输出格式

报告以Markdown格式输出，包含：
- 表格形式的统计数据
- 时间线展示的进度
- 关键指标的可视化
- 具体的文件链接

## 注意事项

- 报告基于实时数据生成
- 可以定期生成报告跟踪进度
- 支持导出为文件保存
